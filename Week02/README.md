# Компилация, Makefile и CМake. Работа с файлове

## Компилация
Най-просто казано, компилацяита е процеса по превръщане на код, разбираем от хората, в машинен език/код, разбираем от компютъра. Както 
можете да се досетите, това не е особено прост процес - той преминава през всички стъпки, показани на диаграмата (не е нужно да 
разбирате всичко за момента). По-важните компоненти засега са:

* Source.cpp - вашата програма, написана на C++ (т.е. разбираемия за хора код)
* Компилатор - превръща human-readable код в машинен. Примери: g++, clang, mingw32, msvc...
* Source.o - машинен код
* Source.exe - изпълним машинен код

![](Week02/resources/compilation-process.png)

### Компилация извън IDE-то (чрез g++ компилатора)
```c++
g++ -c main.cpp      // създава файл с машинен код main.o
g++ main.o           // създава изпълним файл с машинен код a.exe
g++ main.cpp         // прави двете горни операции наведнъж
g++ main.cpp -o main // същото като g++ main.cpp, но кръщава изпълнимия файл main.exe
```

## Build системи
Ръчната компилация на една програма бързо може да стане много трудоемка - най-малкото, че при всяка промяна в съдържанието на даден файл 
трябва да го пре-компилираме, за всеки нов .cpp файл трябва да създадем .exe. Затова съществуват build системи.

Build система е инструмент, който автоматизира процеса на компилация (превръщане на human-readable код в машинен). Той определя **какво** 
трябва да се компилира, **в какъв ред**, и **как** да се свържат всички части заедно.

Примери за build системи са `make`, `ninja`, `MSBuild` (на Visual Studio), `xmake`, `Bazel`

### Къде и защо се използват build системи?
Досега най-вероятно не сте се сблъсвали с build система, тъй като вашето IDE се грижи за всичко, свързано с компилацията. Тогава защо 
изобщо ни е нужно да се занимаваме с тях?

Защото, чрез build системи, кодът ви може да се компилира на различни платформи. Когато ви се наложи да работите в екип, не всеки ще 
ползва същата конфигурация като вас - различни операционни системи (Windows, MacOS, Linux), различни IDE-та (Visual Studio, VSCode, CLion, 
...) и т.н. Build системите осигуряват еднаква компилация на всяка от тези системи, независимо от конкретните им характеристики.

## Какво е Makefile?

Makefile е текстов файл, който автоматизира процеса по компилация. Описва в какъв ред, кои файлове да се 
компилират, с какви флагове, и как да се свържат в изпълним файл. Build системата `make` чете този файл и изпълнява описаните стъпки.

Basic пример:
```c++
output: main.o message.o
    g++ main.o message.o -o output

main.o: main.cpp
    g++ -c main.cpp

message.o: message.cpp
    g++ -c message.cpp
```
## Какво е CMake?

Проблем на `make` файловете е, че в някакъв момент и те самите стават трудоемки за писане (например само за top-level частта на Linux, 
`make` файлът е повече от 2000 реда). Налага се да автоматизираме процеса по автоматизация на компилиране.

Тук се намесва CMake - това е генератор за build системи, който се грижи именно за тази автоматизация. Създава се единен `CMakeLists.txt` 
файл с описание на проекта, а CMake генерира подходящи build файлове за вашата платформа - `Makefile` за Linux/macOS, Visual Studio `.sln` за Windows, Ninja файлове за CLion и т.н.

На практика `CMake` вече се е превърнал в единен стандарт за модерния C++: по подразбиране всеки ползва него, а и самите IDE-та (или 
поне по-популярните) имат интеграция за CMake. Един `CMakeLists.txt` работи за всяка ОС, така че е cross-platform (оттук идва и 'C' в CMake)


## Работа с файлове

Поток (stream) - последователност от байтове данни влизащи в и излизащи от програмата.

![](Week02/resources/streams.png)

Стъпки за работа с поток:
* Създаване на поток
* Свързване с източник/получател на данни (IO устройство)
* Проверка дали потока е отворен
* Работа с потока
* Затваряне и освобождаване на потока

Видове потоци
* входни потоци - istream
* изходни потоци - ostream

![](Week02/resources/hierarchy.png)

Тези потоци се намират във `fstream`

### Основи на потоците
* `istream` има get указател: сочи позиция в потока -> показва откъде ще започне **четенето** при следващата **входна** операция
* `ostream` има put указател: отново, сочи позиция в потока -> показва откъде ще започне **писането** при следващата **изходна** операция
* `istream` има фунцкия `get`, която чете текущия символ и мести get указателя на следващата позиция (абсолютно аналогично за `ostream` и `put` функцията)

#### Позициониране във файл
* **tellg()** - Връща позицията на текущия символ в потока за четене (т.е. връща get)
* **tellp()** - Връща позицията на текущия символ в потока за писане (т.е. връща put)
* **seekg(offset, direction)** - Премества get указателя с **offset** стъпки от **direction** в потока за четене
* **seekg(idx)** - Премества get указателя на позиция idx в потока за четене
* **seekp(offset, direction)** - Премества put указателя с **offset** стъпки от **direction** в потока за писане
* **seekp(idx)** - Премества put указателя на позиция idx в потока за писане
* ignore(n = 1, delim = eof) - Премества get указателя, като прескача следващите n (по подразбиране n = 1) на брой символа или до първото срещане на разделителя **delim** (по подразбиране - краят на файла)
* peek() - Връща следващия символ **без да мести get-указателя**

Възможните стойности за **direction** са:
* std::beg -> сочи началото на файла
* std::curr -> сочи текущата позиция във файла
* std::end -> сочи края на файла

### Примери
Четене от файл
```c++
#include <iostream>
#include <fstream>

int main()
{
   std::ifstream file("preview.txt");

   if (!file.is_open()) { //задължително проверяваме дали файла е отворен
   	return -1;
   }
   
   int a, b;
   file >> a >> b;

   file.close(); // задължително затваряме поток, който сме отворили ние, когато сме приключили работа
}
```

Писане във файл
```c++
#include <iostream>
#include <fstream>

int main()
{
   std::оfstream file("preview.txt");

   if (!file.is_open()) { //задължително проверяваме дали файла е отворен
   	return -1;
   }
   
   int a = 1, b = 2;
   file << a << b;

   file.close(); // задължително затваряме поток, който сме отворили ние, когато сме приключили работа
}
```

Работа със структури
```c++
struct Car {
    char owner[100];
    int price;
};
```
* сериализиране
```c++
std::оfstream file("preview.txt");

   if (!file.is_open()) {
   	return -1;
   }
   
   Car car = {"Pesho", 100};
   file << car.owner << std::endl << car.price;

   file.close();
```

* десериализиране
```c++
std::ifstream file("preview.txt");

   if (!file.is_open()) {
   	return -1;
   }
   
   Car car;
   file.getline(car.owner, 100);
   file >> car.price;

   file.close();
```
