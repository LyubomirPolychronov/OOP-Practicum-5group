# Копиращ конструктор и operator=. Rule of Three (Голяма четворка)


## Копиране на обекти
Често ни се налага да копираме два обекта на даден клас, т.е. да ги приравним с =. Това се нарича operator= (ще учим за оператори по-нататък, засега запомнете, че като говорим за operator= имаме предвид, че приравняваме един обект на друг).
```c++
struct Example {
	double a;
	int b;
	char c;
	char d[100];
	
	// ...представяме си че имаме нужните конструктори...
};

int main() {
	Example e1(4.5, 4, 'а', "word");
	Example e2;
	
	e2 = e1;
	/* вътрешно това което се случва е, че за всяка една член-данна на e2 
	се извиква operator= със съответната член-данна на e1, т.е.
	
	e2.a = e1.a
	e2.b = e1.b
	e2.c = e1.c
	
	при копирането на d се използват  оптимизирани low-level команди на процесора, 
	които ще учите по Операционни системи, засега не ви трябва да знаете,
	важното е, че се получава нещо като:
	for (int i = 0; i < 100; i++) {
		e2.d[i] = e1.d[i]
	}
	
	това е възможно, понеже d е масив със статичен размер, т.е. паметта за него е все една и съща за всяка инстанция на Example 
	*/
}
```
### Проблемът
Но какво става, ако имаме динамична памет? Когато приравняваме указатели ние просто ги караме да сочат към едно и също място в паметта, а не две отделни с еднакво съдържание (това се нарича **shallow copy**)
```c++
class Student {
	char* name;
	
	Student(const char* name) {
		this->name = new char[strlen(name) + 1];
		strcpy(this->name, name);
	}
	
	~Student() { // заделили сме памет за name - трябва и да я освободим
		delete[] name;
		name = nullptr;
	}
	
	void print() const { // принтираме адреса в паметта, на който е записан name
		std::cout << &name << '\n';
	}
};

int main() {
	Student s1("Ivan");
	Student s2;
	s2 = s1; // т.е. s2.name = s1.name
	
	s1.print(); //00D3FA...
	s2.print(); //00D3FA... -> едно и също място в паметта
}
```
Както видяхме, name на s1 и s2 вече сочат едно и също място в паметта. Какво става обаче, когато излезем от scope-а на функцията и се извикат дестукторите? Извикваме деструктора на s2, който зачиства паметта за name, след което идва деструктора на s1 - паметта, която се опитва да достъпи, вече я няма и програмата няма да работи.

### Решението
Вместо да правим **shallow copy**, т.е. просто два указателя да сочат едно и също място, искаме да направим **deep copy** - двата указателя да сочат на различни места в паметта, но с еднакво съдържание. Това означава, че ние сами трябва да си разпишем логиката за копиране.

### Реализация
Разбрахме какво трябва да направим, но как? Нека първо се замислим кога можем да копираме два обекта:
* при създаването на обект искаме той да е копие на друг (щом ще го създаваме, то ще бъде някакъв конструктор -> **копиращ констуктор**) -> имаме една операция: *копиране*
* имаме вече създаден обект, с някакви съществуващи данни, и в последствие искаме да го изравним на друг. Щом вече имаме някаква заделена памет е логично първо да я освободим. Така имаме две операции: *изчистване на заделената памет* и *копиране* -> това се случва при **operator=**

Накрая, когато искаме да изтрием дадените обекти, т.е. да им извикаме **деструкторите**, имаме отново една операция: *изчистване на паметта*

```c++
class Student {
	char* name;
	int year;
public:
	Student(const char* name, int year) {
		this->name = new char[strlen(name) + 1];
		strcpy(this->name, name);
	}
	
	// това е копиращия конструктор - приема референция към обекта, който копираме
	Student(const Student& other) {
		size_t nameLen = strlen(other.name);
		this->name = new char[nameLen + 1];
		strcpy(this->name, other.name);
	
		this->year = other.year;
	}
	
	/* това е operator= -> тъй като това вече не е констуктор, то трявба да имаме тип на връщане на данните, 
	който е Student& (референция от типа на класа, в този пример просто е Student)
	А защо ни е референция? Хубавото на operator= е, че можем да го chain-ваме, т.е. s1 = s2 = s3 = s4 е валиден синтаксис.
	Ще връщаме референция към текущия обект, т.е. *this
	*/
	Student& operator=(const Student& other) {
		/*В случая проверчваме дали не подадем същия обект на operator= (s1 = s1).
		Ако направим нещо такова, първо ще изтрием паметта от this, която е същата като в other, след което като се опитаме да копираме ще копираме вече изтрита памет
		*/
	
		if (this != &other) {
			delete[] name;
			size_t nameLen = strlen(other.name);
			this->name = new char[nameLen + 1];
			strcpy(this->name, other.name);
		
			this->year = other.year;
		}
		
		return *this;
	}

	~Student() {
		delete[] name;
		name = nulltpr;
	}
};
```
Това се нарича **Голяма четворка** или **канонична форма на клас** - *конструктор*, *копиращ конструктор*, *operator=* и *деструктор* (ако го търсите на английски е **Rule of Three**, понеже в чуждоезичната литература не включват конструктора).

Както виждате, имаме доста копиращ се код - можем да изнесем логиката по копиране и изтриване на памет да бъде в две отделни фунцкии, стандартно се наричат *copyFrom* и *free*
Така се получава:
```c++
class Student {
	char* name;
	int year;

	// това са само помощни фунцкии, така че ги оставяме private
	void copyFrom(const Student& other) {
		size_t nameLen = strlen(other.name);
		this->name = new char[nameLen + 1];
		strcpy(this->name, other.name);
	
		this->year = other.year;
	}
	
	void free() {
		delete[] name;
		name = nullptr;
	}
	
public:
	Student(const char* name, int year) {
		this->name = new char[strlen(name) + 1];
		strcpy(this->name, name);
	}
	
	Student(const Student& other) {
		copyFrom(other)
	}
	
	Student& operator=(const Student& other) {
		if (this != &other) {
			free();
			copyFrom(other);
		}
		
		return *this;
	}

	~Student() {
		free();
	}
};
```
Оттук нататък винаги, когато имаме динамична памет в клас, ще използваме **Голяма четворка**/**Rule of Three**. Те са шаблонни - можете да използвате този пример.
